# Roadmap

## Planned

- додати тести для отримання топ останніх записів (при цьому розділити ф-ю на _rows_to_entries та get_recent_entries)
- додати CSRF-захист
- unit тести: побудова рядка для запису (формування row), категорії (немає дублікатів, не порожній)
- винести спільний код у static/theme.js (для теми)
- кнопка для сортування категорій або за популярністю, або за алфавітом
- додати вибір валюти та переведення у грн (окреме поле) по курсу НБУ
- додати Health Checks Path для Render
- при внесенні доходу має перекидати на сторінку для внесення доходів (порожню), а не витрат
- додати можливість меню редагувати категорії для витрат та для доходів
- UI: збільшити ширину кнопок по витратах і доходах, текст має бути по центру
- PWA (Progressive Web App) — додати manifest.json і невеликий service worker, щоб іконка на головному екрані виглядала як справжній застосунок (без адресного рядка браузера) і форма частково працювала офлайн.
- винести отримання останніх записів у окремий асинхронний запит (fetch з JS після завантаження сторінки), щоб форма для введення з'являлася миттєво, а список останніх записів підвантажувався окремо, з невеликою затримкою

## Ideas

- динамічний список категорій (має оновлюватися з врахуванням частоти використання категорій)
- вікно зі статистикою в розрізі останніх 30 днів - сума і розбивка по категоріях
- список категорій має відкриватися окремим вікном збоку
- додати зміну вибору валюти (список доступних валют) та вибору дати для курсу (?)
- автоматично змінювати тему залежно від поточного часу
- версія для десктопу і для смартфона
- додати групи категорій
- Останні N записів на головній сторінці — щоб бачити "ага, я вже вносив каву сьогодні" і мати можливість швидко видалити помилковий запис (видалення через API — трохи складніше, бо gspread працює по індексу рядка).
- локальний буфер на випадок відсутності інтернету — якщо ви заповнюєте форму в метро/ліфті, запит до Sheets впаде. Можна робити чернетку в localStorage браузера і показувати "не збереглось, спробувати знову" з кнопкою повтору, не втрачаючи введені дані.
- Логування помилок — зараз помилки лише показуються у flash і зникають. Варто писати їх у файл/консоль (Render показує логи), щоб розуміти, чи стабільно працює запис.
- Retry-логіка для запису в Sheets — Google API іноді на секунду "зависає" або повертає 429 (rate limit). Зараз при помилці ви просто бачите flash-повідомлення і дані втрачені. Варто додати 2-3 повторні спроби з невеликою затримкою перед тим, як показати помилку.
- Push/Telegram-нагадування — якщо забуваєте вносити витрати, безкоштовний Telegram-бот може раз на день писати "не забудьте внести витрати" з посиланням на форму.
- БД замість google sheets
- тули для AI-агента на основі даних та агрегацій з БД
- UserWarning: Using the in-memory storage for tracking rate limits as no storage was explicitly specified. This is not recommended for production use. See: https://flask-limiter.readthedocs.io#configuring-a-storage-backend for documentation about configuring the storage backend.
